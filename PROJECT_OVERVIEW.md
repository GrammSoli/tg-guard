# SubGuard: Полное описание проекта и архитектуры

**SubGuard** — это полнофункциональное Telegram Mini App (TMA), разработанное для удобного управления подписками, совместной оплаты и отслеживания расходов. Проект переведен на **100% Free-to-Play** модель для быстрого масштабирования аудитории. Монетизация построена на CPA-партнерках, переливе трафика в Telegram-каналы и системе добровольных донатов через **Telegram Stars**.

---

## 🌟 1. Ключевой функционал для пользователей (User-Facing)

### Управление подписками
- **Каталог пресетов**: Более 35 встроенных сервисов (Музыка, ИИ, Игры, Облако и т.д.) с логотипами и фирменными цветами.
- **Кастомные подписки**: Возможность добавить любой свой сервис, если его нет в каталоге.
- **Мультивалютность и Сводка (Summary)**: Визуальный дашборд с расчетом трат (в месяц / в год). Автоматическая конвертация стоимости подписок из разных валют в единую **Базовую валюту** (RUB, USD, EUR, GBP, KZT) по актуальным кросс-курсам.

### Совместные подписки (Shared Rooms)
- Создание "Комнат" для разделения стоимости подписки (например, семейная подписка Spotify на 4-х человек).
- Приглашения через Telegram Deep Links (`t.me/SubGuardBot?start=room_123`).
- Учет оплат: статус "Кто кому сколько должен", кнопка для отправки пинга (напоминания об оплате).

### Smart Push-уведомления и локализация
- **Мультиязычность**: Полная поддержка русского и английского интерфейса как внутри бота, так и в TMA.
- **Умные часовые пояса**: Автоматический (через Web App) и ручной выбор часового пояса (Timezone).
- **Точные уведомления**: Воркер на Go автоматически отправляет напоминание за 24 часа до списания средств в комфортное дневное время по локальному времени пользователя.

### Статус "Donator" (Telegram Stars)
- Пользователи могут добровольно поддержать проект с помощью Telegram Stars. 
- За пожертвования (на любую сумму) пользователь получает эксклюзивный статус и анимированный бейдж ✨ в профиле и на дашборде.

## 🛡️ 2. Админ-панель и CRM (Owner-Facing)

Панель скрыта от обычных пользователей и доступна прямо в TMA только владельцам (указанным в `ADMIN_TELEGRAM_IDS` внутри `.env`).

### Аналитика (Stats)
- **Live Metrics**: Total Users, DAU (Daily Active Users), MAU (Monthly Active Users), количество Donator-ов.
- **Популярность сервисов**: Топ-10 самых добавляемых юзерами подписок (помогает понять, какие партнерские программы искать).

### Трекинг трафика и ROI (Deep Link Generator)
- Генератор рекламных ссылок вида `t.me/SubGuardBot?start=ad_channel`.
- **Сквозная воронка**: 
  - `Bot Starts` -> `TMA Auths` -> `First Subscription` -> `Stars Donation`.

### Умные рассылки (Broadcaster)
- Массовая отправка сообщений по всей базе с поддержкой кнопок.
- **Сегментация**: Возможность исключить из рассылки тех, у кого уже есть подписка на конкретный сервис (защита от спама).
- Авто-перевод: бот сам подставит RU или EN текст в зависимости от локали получателя.

### Монетизация и Глобальные Настройки (Feature Toggles)
- Единый центр управления приложением, работающий без перезагрузки серверов (настройки сохраняются в таблице `AppSettings`).
- **Модуль CPA (Рекомендации)**: Встроенные партнерские (Affiliate) карточки выводятся на главном экране (Dashboard). Админ может в один клик включить или выключить их отображение.
- **Обязательная подписка (Soft-Gate)**: Инструмент для перелива трафика. При включении юзеры не смогут создавать совместные комнаты до тех пор, пока не подпишутся на указанный Telegram-канал (проверяется через Webhook и Bot API). 

### Редактор каталога (Catalog Manager)
- Управление базой пресетов: добавление новых брендов, цветов, валют по умолчанию.
- Поддержка `partner_link` и `promo_text` — возможность зашить реферальную ссылку на сервис и продающий текст для отображения в модуле рекомендаций (CPA).

## 🏗 3. Техническая архитектура

Проект разделен на изолированные микросервисы (Backend и Frontend), общающиеся по REST API.

### Backend (Golang)
- **Фреймворк**: Fiber (высокопроизводительный роутер).
- **База данных**: PostgreSQL (основное хранилище) + Redis (быстрый кэш для валютных курсов).
- **ORM**: GORM (автоматические миграции: `AutoMigrate`).
- **Фоновые процессы (Workers)**:
  - `Notification Worker`: Проверяет подписки батчами (`FindInBatches`) и рассылает алерты без риска OOM (Out of Memory).
  - `Currency Worker`: Ежедневно скачивает актуальные курсы валют из открытых API и складывает в Redis.
- **Интеграция**: Webhook-сервер для обработки событий бота, включая `PreCheckoutQuery` и `SuccessfulPayment` для Telegram Stars.

**Ключевые таблицы БД**:
- `Users` (юзеры, их локаль, timezone, traffic_source_id, базовая валюта, статус донатера).
- `ServiceCatalog` (база брендов с affiliate-ссылками).
- `Subscriptions` (конкретные подписки юзеров).
- `SharedRooms` & `RoomMembers` (совместные оплаты).
- `TrafficCampaigns` (рекламные ссылки и воронка).
- `AppSettings` (глобальные настройки монетизации и Soft-Gate).
- `Donations` (лог успешных платежей в Telegram Stars).

### Frontend (React + TypeScript)
- **Сборщик**: Vite.
- **Стейт-менеджер**: Zustand (глобальные состояния для юзеров, каталога, подписок).
- **Стилизация**: Tailwind CSS + shadcn/ui + Lucide Icons.
- **Интеграция**: `@twa-dev/sdk` для получения безопасных данных от Telegram, Haptic Feedback, открытия ссылок (`openLink`) и платежных форм (`openInvoice`).
- **Роутинг**: React Router (поддержка "страниц" внутри мини-аппа).

## 🚀 5. Развитие и Планы
Текущий проект полностью завершен и готов к релизу (Release Candidate). Дальнейшее развитие может включать:
1. **Экспорт аналитики**: Добавление кнопки выгрузки таблицы воронки трафика в CSV/Excel для маркетологов.
2. **AI Парсинг Чеков**: Возможность прислать боту скриншот выписки из банка для автоматического добавления подписки в TMA.
3. **Геймификация**: Добавление лидербордов (топ донатеров) или достижений (сохранено X рублей).
